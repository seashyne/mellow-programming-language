# Native Mellow vs Lua benchmark — 2026-06-23

## Environment

- Mellow: `Mellow Programming Language 2.9.6 (Full Native C)`
- Mellow binary: `build/standalone-bench-release/Release/mellow.exe`
- Lua: `Lua 5.1.5`
- Lua binary: `D:\Program\ZeroBraneStudioEduPack-2.01-win32\bin\lua.exe`
- OS shell: Windows / PowerShell

## Workload

Both programs compute the same integer sum with a `while` loop:

```text
n = 2,000,000
sum i from 0 to n - 1
expected output = 1999999000000
```

Files:

- `benchmarks/native_vs_lua/sum_loop.mellow`
- `benchmarks/native_vs_lua/sum_loop.lua`

## Timing method

- 10 full process runs per language.
- Measured wall-clock time from process start to process exit.
- This includes source parse/compile/load/startup for both languages.

## Results

| Runtime | Median ms | Average ms | Min ms | Max ms | Iterations/sec by median |
| --- | ---: | ---: | ---: | ---: | ---: |
| Mellow native C Release | 244.68 | 253.28 | 227.47 | 305.98 | ~8.17M/s |
| Lua 5.1.5 | 8.08 | 8.34 | 7.72 | 9.58 | ~247.52M/s |

Lua 5.1.5 is about **30.3x faster** on this tight arithmetic loop by median time.

## Notes

This benchmark stresses VM dispatch, local variable load/store, integer arithmetic,
and loop branching. It does not measure I/O, package loading, native built-ins,
or host integration.

The current Mellow native VM still deep-copies values on many local loads and
uses a generic tagged-value dispatch path. The biggest likely wins are:

1. avoid deep-copying scalar locals on `LOAD_LOCAL`;
2. add borrowed/owned value semantics for stack reads;
3. specialize numeric loops and arithmetic opcodes;
4. add bytecode peephole optimization for `while i < n`;
5. optionally add a register VM or threaded dispatch for hot paths.
